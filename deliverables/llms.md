# AI/LLM Native

## Dependencies

- [Community alignment](/deliverables/alignment.md)
- [Developer HUB](/deliverables/developer-hub.md)

## Description

AI/LLM-assisted development is here to stay, so we need to ensure our ecosystem is prepared for that.

We'll make the Cardano developer ecosystem AI/LLM-native by ensuring all documentation and tooling is optimized for consumption by LLMs and AI-powered development tools. This includes:
- **Making documentation machine-readable through standardized LLM.txt files:** Well help add `llms.txt` files to the documentation of commonly used resources. Not only the [developer HUB](/deliverables/developer-hub.md) presented in this strategy, but also to common tooling and libraries created and maintained by other teams (e.g., Aiken, Tx3, MeshJS, etc.).
- **Providing integrated access to docs through an MCP servers:** Same idea as previous point but for MCP servers.
- **Creating Agent Skills for common development workflows:** We'll create a set of Agent Skills for common development actions (e.g., testing contracts, updating dependencies, etc.) for the most common tools and workflows.
- Other measures that might become apparent during the implementation of this plan.

As AI-assisted coding becomes the norm, ensuring Cardano's ecosystem is first-class in AI tooling will be a significant competitive advantage for developer onboarding and productivity.


## Outcome

The outcome is that developers will face no friction to use their LLMs to assist their development on Cardano.
