# OpenZeppelin-like Library

## Dependencies

No dependencies

## Description

We'll design and implement a library of standardized and reusable smart contracts — both on-chain and off-chain — with an initial emphasis on building blocks and DeFi primitives. Inspired by OpenZeppelin's role in the EVM ecosystem, this library will provide battle-tested, ready-to-use contract implementations that developers can use as either building blocks or complete implementation for their applications. 

The deliverable includes creating the infrastructure and website for the library and shipping _at least_ 5 ready-to-audit smart contracts (both on-chain and off-chain). For example, Vesting, Programmable Tokens, DEX (Decentralized Exchanges), Swap, Lending, and other contracts.

> NOTE: It's important to note the difference between this library and libraries like Anastasia Labs's [design-patterns](https://github.com/Anastasia-Labs/design-patterns) and [Vodka](https://github.com/sidan-lab/vodka). While `vodka` contains small on-chain utility functions and `design-patterns` contains fully-fledged on-chain generic patterns, our library's level of abstraction will be at the use case level and provide both on-chain and off-chain.

## Outcome

By providing canonical implementations of common smart contract use cases, this library will significantly reduce the time and risk involved in building on Cardano.
